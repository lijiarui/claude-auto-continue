Claude Auto Continue

不用再手动点 Continue。

一个 Chrome 扩展。它在 claude.ai 上看到 Continue / 继续生成 按钮就替你点掉。安全、本地、可关。

装上它 在 GitHub 看源码 →
✓ 自动点击

真实场景。看到这个就替你点，对话不中断。

它解决什么

claude.ai 单轮工具调用约 20 次就会撞上 Claude reached its tool-use limit for this turn.，要手动点 Continue 才能继续。回复被 max_tokens 截断、长会话警告，也都是同一类提示。

做数据分析、跑 MCP、连飞书 / Drive 的人，一天踩几十次。这个扩展守在页面上，按钮一出现就替你点。

它怎么不出错

Allowlist

只点匹配的按钮

必须精确等于或前缀是 Continue / 继续 等才点。

Exclude

排除危险词

含 Cancel / 删除 / Continue with Google 的按钮直接跳过。

节流

防止抽风点击

两次点击至少 1.5s 间隔，同一按钮 3s 内只点一次。

本地

零网络请求

不上传数据，不调外部 API。配置存 chrome.storage.local。

怎么安装

商店审核中。在那之前，可以用开发者模式装：

  1. 克隆仓库：git clone https://github.com/lijiarui/claude-auto-continue.git
  2. 打开 Chrome 的 chrome://extensions，右上角开启「开发者模式」
  3. 点「加载未打包的扩展程序」，选 claude-auto-continue/ 目录
  4. 打开 claude.ai，开始聊天即可

常见问题

它会自动点登录 / 删除按钮吗？
不会。Allowlist + exclude 双重过滤。Continue with Google、Cancel、删除 这类全部跳过。
会不会上传我的对话内容？
不会。扩展只读按钮文本来判断要不要点，不读对话，不发任何网络请求。
突然不工作了怎么办？
大概率是 claude.ai 改了按钮文案。点扩展图标，把新文案加到「匹配按钮文本」里。或在 GitHub 提 issue。
能不能在 ChatGPT / Gemini 上用？
当前版本只注入 claude.ai。如果有需求可以在 GitHub 提 issue。
